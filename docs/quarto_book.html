Guía de Proyecto Quarto Book

Este documento describe, en términos generales, cómo está organizado el proyecto Quarto de tipo book, cuándo y cómo usar los distintos sistemas de referencias cruzadas, y cómo integrar citas bibliográficas con references.bib.

Estructura general del proyecto

El archivo _quarto.yml controla la configuración principal del libro. En tu caso:

project:
  type: book

book:
  title: "Análisis del Mercado Inmobiliario en Ames, Iowa"
  subtitle: "Estadística Descriptiva, FCEA-UdelaR"
  author:
    - Lucas Rodriguez
    - Nahir Silva
  date: "6/28/2025"
  chapters:
    - index.qmd
    - chapters/01-introduction.qmd
    - chapters/02-data-description.qmd
    - chapters/03-data-cleaning.qmd
    - chapters/04-transformations.qmd
    - chapters/05-eda.qmd
    - chapters/06-modeling.qmd
    - chapters/07-clustering.qmd
    - chapters/08-conclusions.qmd
  appendices:
    - chapters/09-appendix.qmd
    - references.qmd

bibliography: references.bib
output-file: "Informe_Ames_Housing.pdf"

format:
  html:
    self-contained: true
    toc: true
    toc-depth: 3
    code-fold: false
    code-tools: true
    code-summary: "Mostrar código"
    theme:
      - cosmo
      - brand
  pdf:
    documentclass: scrartcl
    include-before-body: "titlepage.tex"
    linestretch: 1.2
    geometry: margin=1in

execute:
  echo:    true
  warning: true
  message: false
  • chapters: define los .qmd que forman el cuerpo numerado.
  • appendices: agregan contenido no numerado (Apéndices y Referencias).
  • bibliography: apunta al archivo BibTeX con tus fuentes.

Referencias cruzadas: dos sistemas

En Quarto Book existen dos maneras de crear enlaces internos:

Anclas en encabezados ({#id})

  • Uso: para referenciar secciones o apartados de texto.

  • Implementación: añadís {#mi-seccion} a un encabezado de nivel 2 o 3:

    ## Resultados del EDA {#sec-resultados}

  • Enlace:

    Ver la sección [Resultados del EDA](#sec-resultados).

  • Características:

    • No genera numeración automática de la referencia.
    • Es un hyperlink estándar en HTML y PDF.

Cross-referencing con captions y @id

  • Uso: para referenciar objetos numerados — tablas, figuras, ecuaciones, bloques de código.

  • Implementación:

    1. Chunk nombrado:

       ```r
 knitr::kable(tabla_barrio)

      ```

    2. Caption debajo:

      Tabla: Distribución completa de viviendas por barrio

    3. Referencia:

      Consultar la tabla @tbl-barrio para más detalles.

  • Características:

    • Numeración automática según el capítulo o Apéndice (p. ej. “Tabla A.1”).
    • Funciona en HTML, PDF y DOCX.
    • Permite generar índices de figuras/tablas si se activa en el YAML.

¿Cuándo usar cada sistema?

Tipo de contenido Sistema recomendado
Apartados, secciones de texto Anclas en encabezados {#id}
Tablas, figuras, ecuaciones Captions + chunks nombrados @id
  • No mezclar ambos métodos para el mismo objeto.
  • Es válido usar ambos en un mismo proyecto, siempre en su contexto apropiado.

Bibliografía con references.bib

  1. Archivo BibTeX:

    • Guarda en references.bib todas tus entradas:

      @article{ames_decock_2011,
  author  = {De Cock, Dean},
  title   = {Ames, Iowa: Alternative to the Boston Housing Data},
  journal = {Journal of Statistics Education},
  year    = {2011},
}

  2. Citas en el texto:

    • Usá [Texto de apoyo @clave] o solo [@clave] en cualquier .qmd:

      Según De Cock [@ames_decock_2011], este dataset…

  3. Listado de referencias:

    • Conservar references.qmd como Apéndice final:

      # Referencias {.unnumbered}

::: {#refs}
:::

    • Quarto compilará automáticamente la sección con todas las entradas citadas.

Flujo de trabajo resumido

  1. Editar _quarto.yml y confirmar capítulos y appendices.

  2. Crear cada capítulo en chapters/ siguiendo la estructura propuesta.

  3. Nombrar chunks de tablas/figuras y agregar captions + @id.

  4. Citar con [@clave] donde corresponda.

  5. Referenciar con @tbl-mi-tabla o [texto](#mi-seccion).

  6. Compilar con:

    quarto render
